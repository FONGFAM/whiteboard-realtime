1. PROJECT OVERVIEW
================================================================================

A collaborative realtime drawing platform enabling multiple users to draw 
simultaneously on shared virtual whiteboards. Built with Spring Boot backend 
(WebSocket/STOMP) and HTML5 Canvas frontend.

Key Achievement: Implemented full realtime sync, undo/redo per-user, and 
multi-board support in ~2000 lines of code.

4. ARCHITECTURE HIGHLIGHTS
================================================================================

BACKEND DESIGN:
  • WebSocketConfig: STOMP broker + SockJS endpoint setup
  • DrawingController: Message handlers for stroke/sync/action
  • BoardService: Business logic (undo/redo, state management)
  • In-memory HashMap: Fast state storage (no DB overhead)
  • SimpleBroker: Broadcasts messages to all subscribers

FRONTEND DESIGN:
  • Canvas Engine: Pointer events → stroke accumulation → local render
  • STOMP Client: Connection → subscription → message send/receive
  • Throttling: Reduce network traffic via sampling
  • Redraw Loop: Re-render canvas on state change

MESSAGE PROTOCOL:
  • /app/stroke/{boardId}: Send drawing
  • /app/sync/{boardId}: Request full state
  • /app/action/{boardId}: Undo/redo/clear
  • /topic/board/{boardId}: Broadcast channel

9. KNOWN LIMITATIONS (MVP)
================================================================================

⚠️  NO PERSISTENCE:
  - Board state lost on server restart
  - No database backend
  (Solution: Add MongoDB in v2.0)

⚠️  NO AUTHENTICATION:
  - Any user can join any board
  - No user login required
  (Solution: Add JWT in v2.0)

⚠️  SINGLE INSTANCE:
  - Cannot scale to multiple servers
  - No cross-instance communication
  (Solution: Add Redis pub/sub in v2.0)

⚠️  LIMITED TOOLS:
  - No eraser tool (use Clear instead)
  - No shapes, text, or layers
  (Solution: Add in v2.0)

⚠️  NO RECORDING:
  - Can't replay drawing sessions
  (Solution: Add in v2.0)
