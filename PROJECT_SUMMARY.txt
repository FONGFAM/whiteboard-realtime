================================================================================
REALTIME WHITEBOARD - PROJECT SUMMARY (v1.0.0)
================================================================================

PROJECT: Multi-user realtime drawing app with Spring Boot WebSocket backend

TECH STACK:
 Backend: Java 17, Spring Boot 3.2, WebSocket (STOMP/SockJS), Maven
 Frontend: HTML5 Canvas, JavaScript ES6+, SockJS, CSS3

================================================================================
FEATURES IMPLEMENTED
================================================================================

CORE:
 - Realtime stroke sync (<100ms latency)
 - Per-user Undo/Redo
 - Room-based boards (join by ID)
 - Full state sync on client join
 - Color picker, line width, clear button
 - Auto-reconnection, user persistence

TECHNICAL:
 - Message protocol: /app/stroke, /app/sync, /app/action
 - Broadcast via STOMP SimpleBroker
 - In-memory HashMap state (fast, no DB overhead)
 - Point throttling (~60fps)
 - SockJS WebSocket fallback

================================================================================
KNOWN LIMITATIONS (MVP)
================================================================================

- No database persistence (state lost on restart)
- No authentication (any user can join any board)
- Single instance only (no multi-server scaling)
- Limited tools (draw, undo, redo, clear only)
- No recording/replay
